# SODA - 프로젝트 맞춤 데이터 자원 통합 추천 플랫폼

> 프롬프트 한 줄이면, 프로젝트에 맞는 데이터셋과 Open API를 함께 찾고 바로 검토할 수 있게 돕는 서비스

---

## 문제 인식

데이터 기반 프로젝트를 시작할 때 개발자는 여러 사이트를 직접 돌아다니며 데이터셋과 Open API를 찾아야 한다. 찾은 뒤에도 다음 판단이 어렵다.

- 내 프로젝트 목적에 정말 맞는가
- 데이터 구조와 품질이 usable한가
- API가 실제 서비스에 붙일 만큼 안정적인가

SODA는 이 탐색과 판단 비용을 줄이는 것을 목표로 한다.

---

## 해결 방식

사용자가 프로젝트 목적을 자연어로 입력하면:

1. 수집된 데이터셋/Open API 메타데이터에서 후보를 검색하고
2. `pgvector` 기반 의미 검색으로 관련 후보를 추리고
3. LLM이 실제 검색 결과를 바탕으로 추천 이유와 활용 가이드를 생성한다

즉, **RAG 기반 추천 + 구조 분석 + 적합도 판단**을 한 번에 제공한다.

---

## 핵심 기능

- 프롬프트 기반 데이터셋/Open API 통합 검색
- 프로젝트 목적별 적합도 점수 제공
- 데이터 구조 및 API 스펙 자동 분석
- 데이터 신뢰도 검증 정보 제공
- 프로젝트 가이드와 데이터 조합 제안

---

## 선택한 시스템 구조

```text
외부 소스
  -> Celery Beat
  -> RabbitMQ
  -> Celery Worker / Crawler
  -> PostgreSQL + pgvector / S3
  -> FastAPI RAG
  -> 웹 UI
```

### 역할 분리

- `RabbitMQ`: 작업 브로커
- `Celery`: 수집, 임베딩, 재색인 비동기 실행
- `Redis`: 캐시, rate limit, lock, 짧은 상태값
- `PostgreSQL + pgvector`: 메타데이터와 벡터 검색
- `S3`: 원본 파일과 대용량 산출물 저장

### 서버 구성

- `S1`: 단일 Nginx, `prod active`, `dev active`, `dev PostgreSQL / Redis / MinIO`
- `S2`: RabbitMQ, Celery Worker/Beat, crawler, Jenkins, monitoring
- 현재 구조는 `S1` 단일 ingress를 사용

### 범위에서 제외한 것

- `HDFS`
- `Spark`
- `prod` 자동 failover
- `dev` 자동 failover

## 기술 스택

- Frontend: React, TypeScript
- Backend: Spring Boot, FastAPI
- Async: Celery, RabbitMQ
- Cache/State: Redis
- Storage: PostgreSQL + pgvector, S3/MinIO
- Infra: Docker Compose, Nginx, Jenkins, Prometheus, Grafana, Loki

## 차별점

- 데이터셋과 Open API를 한 곳에서 함께 추천
- 실존 데이터 기반 RAG 추천으로 할루시네이션 최소화
- 구조 분석과 적합도 판단을 동시에 제공
- 비동기 수집 파이프라인으로 최신성 유지

---

## 기대 효과

SODA는 데이터 탐색 시간을 줄이고, SSAFY 교육생이 더 빠르게 프로젝트를 시작할 수 있게 하는 공용 데이터 탐색 플랫폼을 목표로 한다.
